Smart Campaigns

Automated text messaging system for sales teams — built at Enzy.

Description

At Enzy, I built the Smart Campaigns system — an automated text messaging platform that lets sales teams send the right message to the right person at the right time, all tied to real appointment and scheduling data. A manager sets up a campaign once, and the system handles everything from there: sending reminders before appointments, follow-ups after, confirmations at creation, and recurring check-ins — all via SMS through Twilio.

The system supports multiple campaign types (leads, recruits, customers, and custom) and multiple delivery methods (text, email, and in-app notifications). Contacts can be enrolled manually or automatically when an appointment is created. The entire system is multi-tenant, with each sales organization's campaigns, subscriptions, and interactions fully isolated at the database level.

The backend is Java with a DAO-based persistence layer on AWS Aurora MySQL, and the frontend is an Angular/Ionic hybrid mobile application with a multi-step campaign builder.

Note: This was built as part of my work at Enzy. Source code is proprietary and not publicly available.

Trigger-Based Messaging Architecture

The core of Smart Campaigns is the trigger system. Each campaign contains a set of triggers, and each trigger defines three things: when to send, who to send to, and what to send. The "when" is anchored to appointment events:

  • Before Appointment — Send a reminder X hours or days before the scheduled time.
  • After Appointment — Send a follow-up X hours or days after.
  • At Appointment Creation — Send an immediate confirmation when the appointment is booked.
  • Delay — Send a fixed time after the contact is enrolled in the campaign.
  • Recurring — Send on a repeating schedule.

The "who" is equally flexible — a trigger can target the customer, the sales rep who set the appointment, the closer, or the setter. This means a single campaign can coordinate messages across multiple people involved in the same deal. Each trigger carries its own message content and delivery method, and can optionally include the rep's digital business card or an appointment reschedule link.

Scheduled Execution & Smart Rescheduling

A Quartz-based job runs periodically across all active workspaces, evaluating every active subscription against its campaign's triggers. For each trigger, the system calculates the exact send time based on the anchor event and time offset. If the message is more than a few minutes out, it's scheduled directly with Twilio for future delivery. If it's imminent, it's sent immediately. Each sent or scheduled message is recorded as an interaction with its Twilio message ID for status tracking.

A key design challenge was handling contact changes after messages are already scheduled. When an appointment is rescheduled or a contact's information is updated, the system needs to cancel old scheduled messages and recalculate new send times. This is managed through an update queue with a locking mechanism — when a contact update comes in, the system acquires a lock, deletes the old scheduled Twilio messages, recalculates trigger times against the new appointment data, schedules fresh messages, and releases the lock. This prevents race conditions where concurrent updates could result in duplicate or orphaned messages.

Twilio Integration & Opt-Out Compliance

Each workspace has its own dedicated Twilio phone number. Outbound messages are sent or scheduled through Twilio's API, and status updates (delivered, failed, etc.) flow back through a webhook callback that updates the interaction record in real time.

When a contact replies, the system checks for opt-out keywords (STOP, CANCEL, UNSUBSCRIBE, and others) and automatically marks them as opted out, preventing any future messages. For non-opt-out replies, the system sends back the rep's digital business card and notifies the campaign creator — turning an automated text into a warm conversation handoff.

Contacts can also reschedule appointments directly from a link in the text message. The reschedule flow lets them select a new time or confirm attendance, and the system notifies the sales rep of the response — keeping the entire interaction within the SMS channel without requiring the contact to download an app or visit a portal.

Interaction Tracking & Analytics

Every message in the system is tracked as an interaction record with a status lifecycle: scheduled, sent, delivered, failed, opt-out, invalid number, or unsubscribed. This gives managers full visibility into campaign performance — which messages went out, which failed, and which contacts opted out. The interaction history is viewable per subscription, making it easy to see the complete message timeline for any individual contact across all their campaign enrollments.

Campaign Builder

The frontend uses a multi-step wizard built in Angular/Ionic. The first step captures campaign basics — name, type, and whether it should be shared company-wide or kept private. The second step is the trigger editor, where managers add message rules with the anchor event, timing, recipient, delivery method, and content. The third step configures dispositions — mapping campaign outcomes to the organization's existing status codes. The builder validates that every trigger has complete content before allowing activation, preventing campaigns from going live with missing messages.
