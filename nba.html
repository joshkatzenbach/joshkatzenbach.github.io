NBA Deep Neural Network

Predictive deep neural network using Python's PyTorch package.

Description

The overall goal of this project was to produce a program capable of predicting the outcome of NBA basketball games with greater accuracy than a human could do with the simple stats that are available. More specifically, I wanted to be able to predict the games based off of the stats that were available at halftime of each game. My approach to solving the problem was using a deep neural network that would learn from NBA games of the past in order to identify trends that may be difficult for humans to observe.

Data

I designed a python program that scraped data from basketballreference.com (while respecting rate limits). Using previous basketball knowledge, I identified 14 different statistics that would be available both at halftime and at the end of each game. I avoided the more advanced stats that are created by various combinations of the simple counting stats because a well designed neural network should have the capacity to see the relationships between the simple stats without human intervention. In fact, a neural network should be able to come up with weighted relationships that are even more accurate than the relationships that humans can identity on their own. I had the program scrape that data from over 30,000 NBA games dating back to the year 2000. I also calculated season averages for each team up until the game that they were currently playing - we couldn't use total season averages because that data wouldn't be available in real life dues to those games not being played yet.

Model

The data used here was entirely tabular with no spatial relationships, thus using any version of a convolutional neural network would not be effective. Instead, a fully connected network was utilized. Various different modifications were made to both the data and the network itself in order to optimize the models. These modifications included:
  • Number of Hidden Layers - Anything above 3 hidden layers resulted in overfitting.
  • Loss Function - While a 2 output final score was more intuitive, using one output representing score difference was most effective.
  • Data Normalization - Normalizing each data point across all trials improved both accuracy and stability.
  • Batch Normalization - Adding a batch normalization (activations across a batch) layer to the network drastically improved accuracy.
  • Varied Data Sets - First quarter data was omitted in some trials. This seems to have little to now effect on the outcomes. Using temporal data, which was aquired by calculating averages from the previous ten games,resulted in the most effective runs.
