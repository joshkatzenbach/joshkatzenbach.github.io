Chess Server

Fully functional chess server using custom APIs, JSON, WebSockets, and an SQL database.

Description

This project consists of both the client side and the server side of a chess game. Clients are able to connect to register as a user, create games, and join games as both players and spectators. The complete set of chess rules is implemented into the game. To the right is an image that shows how the chess board would appear within the player's terminal.

Custom RESTful APIs

7 different custom APIs were designed for this project, allowing the client to easily communicate with the server. These APIs used the RESTful architectural style, utilizing DELETE, PUT, GET, and POST types. These APIs are utilized primarily pregame. They carry authorization tokens, allowing the server and the client to keep track of the current session. These APIs include:

  • Register
  • Login
  • Create Game
  • List Game
  • Clear
  • Join Game
  • Logout

WebSockets

Once a game had been joined, the client and server would create a WebSocket connection. This allowed for bi-directional instantaneous communication between the two. This was essential so that each player (or spectator) could receive board updates as soon as another move had been made.

DataBase

The server utilized an SQL database in order to store player data. The server used Database Access Objects (DAO) in order to securely and safely read and right data. This database utilized several different tables, among which were:
  • Current Sessions - Kept track of usernames and authorization tokens to manage open sessions.
  • Users - Stored user data, including usernames, passwords, and emails.
  • Games - Stored data about each game, including the players, the board state, and the game's status.

    • Complex data structures, such as board state, were communicated through RESTful APIs and WebSockets using JSON.
